- BNF문법 유도방법
	- ```apl
	  S -> bAS
	  S -> c
	  A -> SS
	  A -> a
	  ```
	- 비단말기호 : 대문자, 바꿀수 있음
	- 단말기호: 소문자, 못바꿈
	- 시작은 S부터시작
	- bccc를 좌단유도방법
		- S -> bAS : 시작
		- S -> bSSS : A -> SS
		- S -> bcSS : S -> c
		- S -> bccS : S -> c
		- S -> bccc :  S -> c
	- bccc를 우단유도방법
		- S -> bAS : 시작
		- S -> bAc : S -> c
		- S -> bSSc : A -> SS
		- S -> bScc :  S -> c
		- S -> bccc :  S -> c
	- First(A)
		- A로 시작하여 가장먼저 나타날 수 있는 단말기호의 집합
		- A -> SS : S가 나오기 때문에 First(S)먼저 계산
			- S -> bAS : b
			- S -> c : c
			- b, c 가됨
		- A -> a : a
		- b,c,a 가됨
	- Follow(A)
		- A뒤에 올 수 있는 단말기호 또는 종결기호의 집합
			- (A→ 비단말기호, 비단말기호 -> A단말기호) 패턴의 경우
		- S → bAS
			- A뒤에 S가옴, First(S)를 계산
		- A -> SS
			- A뒤에 S가옴, First(S)를 계산
- 모호한 문법
	- 하나의 문자열(토큰 시퀀스)에 대해 둘 이상의 유도 트리(파스 트리)가 존재하는 문법
- 터미널 문자열
	- 문법 규칙을 통해 더 이상 분해하거나 대체할 수 없는 문자열
	- ex) `abc`, `id + id`, `a * b`
- SLR 파싱표
	- 동작칸
		- S: shift, 다음 상태로 이동
		- r: reduce, 특정규칙 축약
		- acc: accept, 수락
	- GOTO칸
		- 비단말 기호를 만났을때 이동해야 되는 상태
